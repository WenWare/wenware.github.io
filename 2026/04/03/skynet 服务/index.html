Skynet 服务启动机制

概述

Skynet 服务作为核心组件，主要负责处理客户端请求、服务间通信等业务功能。

在 Skynet 中，总共分两种服务，一个是 C 服务，一个是 Lua 服务，每个服务都有自己的服务 skynet_context。skynet_context 是 Skynet 中用来维护服务桩体的重要结构体，所有关于服务的操作都是通过 skynet_context 来完成的，例如查找指定服务都是通过里面的 handle 来找到对应的 skynet_context 的。

那么这两种服务都是怎么启动的呢？

C 服务启动机制

C 服务一般在编译成 .so 文件，然后在 Skynet 启动时加载或者在运行时动态加载。

动态库格式要求

这种 C 服务都有固定的文件命名以及函数命名格式（例如 service_logger.so、service_snlua.so 动态库命名），以及库中必须实现以下四个函数：

  • xx_create
  • xx_init
  • xx_release
  • xx_signal

以便在使用 dlopen 进行加载的时候能够正确找到对应的函数。

启动流程

启动或者运行时动态加载时会触发 skynet_context_new(const char * name, const char *param) —— skynet_server.c，在里面会调用到 skynet_module_query(name) —— skynet_module.c。

该函数首先会查询模块是否已经加载过，如果没有加载则会通过 dlopen 实现运行时加载动态库获取到模块对应的动态库的句柄，最终根据句柄调用 dlsym 获取到对应的函数指针，最终实现 C 服务的启动。

在获得动态库的模块后，会调用 skynet_handle_register(struct skynet_context *ctx) 进行 handle 的注册获得一个服务的专属 ID，后续根据这个 ID 进行通信，最后会对这个 skynet_context 对象进行必要的初始化（例如消息队列生成、模块的初始化逻辑）。

Lua 服务启动机制

Lua 服务一般在启动的时候执行 main.lua 文件时，执行 skynet.newservice(name, param) 来启动 Lua 服务。

启动步骤

  1. 服务创建请求：在执行 skynet.newservice(name, param) 最终会在 skynet.launch(...) —— manager.lua 中调用到 lua-skynet.c 里的 command 函数，然后会调用到 skynet_server.c 下的 skynet_command 执行 cmd_launch 函数，这里会执行 skynet_context_new("snlua", "clusterd") 进行 C 服务类型的 snlua 服务的启动。

  2. Snlua 服务初始化：在 snlua 服务启动完毕后，skynet_module_instance_init(mod, inst, ctx, param) 进行 Lua 服务的初始化。在里面调用到 snlua_init 函数，这里会将 launch_cb 当做第一次回调的 cb 函数，snlua_init 结束后会向 lua 服务发送第一条信息，这时已经算启动完成了。

  3. 首次消息处理：在 worker 线程中，执行服务的第一个消息时，会分发到 launch_cb 函数，这里先通过 skynet_callback(context, NULL, NULL) 将服务的 cb 设置为 null，避免重复处理，然后会在 init_cb 中执行 lua_State 的初始化，到这里，该 lua 服务是没有 callback 函数的。

  4. Lua 文件加载：在 init_cb 中，会通过 newservice 的 name 参数，使用 lualoader 的 lua 加载器进行对应 lua 文件的加载，最终使用 lua_pcall 来执行该 lua 文件。

  5. 回调函数注册：在 lua 文件中，会执行到 lua 代码 skynet.start(function() end)，最终执行到 C 代码 callback —— lua-skynet.c，这里会将 skynet.dispatch_message 这个 lua 函数注册到 registry[_cb] 中，在普通模式下，会调用 skynet_callback(context, gL, _cb) 将 lua-skynet.c 下的 _cb 函数设置为该 lua 服务的回调函数。

  6. 消息处理：当有新的消息到达时，在 worker 线程中会调用 ctx->cb(ctx, ctx->cb_ud, type, msg->session, msg->source, msg->data, sz) 从而触发到 lua-skynet.c 中的 _cb 函数，最终调用到 skynet.dispatch_message 进行业务处理。
